Validate the entire Claude Code setup and verify all components are working correctly.

Usage: `/self-test [--fix]`

Options:
- `--fix` - Attempt to automatically fix detected issues

Execute comprehensive validation across all Claude Code components.

## Validation Steps

Run each validation step sequentially, collecting issues. At the end, display all findings and optionally apply fixes if `--fix` flag is provided.

### 1. Directory Structure

Check core directories exist:
- `.claude/core/agents`
- `.claude/core/commands`
- `.claude/examples/skills`
- `.claude/examples/hooks`
- `.claude/docs`

**Auto-fix**: Create missing directories with `mkdir -p`

### 2. Settings Validation

Verify `.claude/settings.json` exists and is valid JSON.

```bash
node -e "JSON.parse(require('fs').readFileSync('.claude/settings.json'))"
```

**Auto-fix**: Create default settings.json if missing

### 3. Core Agents

Verify expected agents exist:
- `security-auditor.md`
- `database-architect.md`
- `api-builder.md`

Check YAML frontmatter is valid in each agent file.

**Auto-fix**: Cannot auto-fix missing agents (user must install)

### 4. Core Commands

Verify expected commands exist:
- `build-safe.md`
- `sync-types.md`
- `db-migrate.md`
- `health-check.md`
- `commit.md`
- `create-pr.md`
- `release.md`
- `enable-hook.md`
- `self-test.md` (this file)

**Auto-fix**: Cannot auto-fix missing commands

### 5. Skills Detection

Count skills in `.claude/core/skills/` and `.claude/examples/skills/`.

List enabled skills by category:
```bash
find .claude/core/skills -name "skill.md" -o -name "SKILL.md" | wc -l
```

**No fix needed** - informational only

### 6. Hooks Configuration

Check if hooks are configured in settings.json:
```bash
node -e "
const settings = JSON.parse(require('fs').readFileSync('.claude/settings.json'));
const hooks = settings.hooks?.PostToolUse || [];
console.log(hooks.length > 0 ? 'Hooks enabled' : 'No hooks configured');
"
```

**Auto-fix**: Offer to enable quality-focused preset

### 7. Framework Detection

Detect framework from package.json if it exists:
```bash
if [ -f package.json ]; then
  if grep -q "\"next\"" package.json; then
    echo "Framework: Next.js"
  elif grep -q "\"react\"" package.json; then
    echo "Framework: React"
  elif grep -q "\"bun\"" package.json; then
    echo "Framework: Bun"
  else
    echo "Framework: Unknown"
  fi
else
  echo "No package.json found"
fi
```

**No fix needed** - informational only

### 8. Environment Configuration

Check if `.env.example` exists (generated by setup.sh):
```bash
if [ -f .env.example ]; then
  echo "✓ .env.example found"
else
  echo "⚠ .env.example missing (run setup.sh to generate)"
fi
```

**Auto-fix**: Cannot generate without setup.sh context

### 9. Git Integration

Check if repository is a git repo and verify .gitignore has Claude entries:
```bash
if git rev-parse --git-dir > /dev/null 2>&1; then
  if [ -f .gitignore ]; then
    if grep -q ".claude/settings.local.json" .gitignore 2>/dev/null; then
      echo "✓ Git configured correctly"
    else
      echo "⚠ .gitignore missing Claude entries"
    fi
  fi
fi
```

**Auto-fix**: Add recommended .gitignore entries

### 10. Plugin Configuration

Check if `.claude/plugin/plugin.json` exists and is valid:
```bash
if [ -f .claude/plugin/plugin.json ]; then
  node -e "JSON.parse(require('fs').readFileSync('.claude/plugin/plugin.json'))"
fi
```

**No fix needed** - informational only

### 11. Documentation

Verify key documentation files exist:
- `.claude/docs/README.md`
- `.claude/docs/quickstart.md`
- `.claude/docs/customization.md`

**No fix needed** - informational only

### 12. Presets Available

List available presets:
```bash
if [ -d .claude/presets ]; then
  ls -1 .claude/presets/*.json 2>/dev/null | xargs -n1 basename -s .json
fi
```

**No fix needed** - informational only

### 13. Health Summary

Display summary of:
- Total agents (core + examples)
- Total commands (core + examples)
- Total skills (core + examples)
- Total hooks enabled
- Framework detected
- Git status

## Auto-Fix Actions

If `--fix` flag is provided and issues are found, apply fixes:

1. **Missing directories**: Create with `mkdir -p`
2. **Missing settings.json**: Create default configuration
3. **Missing .gitignore entries**: Append recommended entries
4. **Other issues**: Display manual fix instructions

After applying fixes, re-run validation to confirm.

## Implementation Notes

- Use bash scripts for file checks and JSON parsing with Node.js
- Collect all issues before displaying
- Color code output (✓ green, ⚠ yellow, ✗ red)
- Keep validation fast (< 5 seconds total)
- Don't modify files without --fix flag
- Provide actionable fix instructions for non-fixable issues
